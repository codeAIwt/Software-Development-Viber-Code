## 3. 问题详细分析

### 3.1 代码Bug（需修复）

#### Bug #1: 销毁房间后仍可被查询

**严重程度**: 高

**问题位置**: `services/room_service.py` 第589-639行 `destroy_room`函数

**问题描述**:
`destroy_room`函数执行后将房间状态设置为"closed"，但没有删除房间的meta数据，导致`get_room_info`仍能通过Redis查询到房间信息并返回200，而非预期的404。

**问题代码**:
```python
# room_service.py 第619-627行
# 更新房间状态为closed
pipe.hset(
    cache.room_meta_key(room_id),
    mapping={
        "current_people": "0",
        "status": "closed",  # 只更新状态，没有删除meta
        "updated_ts_ms": str(now_ms),
    },
)
```

**修复建议**:
在`destroy_room`中，当房间状态设置为"closed"后，应该删除房间的meta数据，或者修改`get_room_info`对"closed"状态返回404。

**对比**: `leave_room`函数在房间无人时（new_status == "closed"）会删除房间meta：
```python
# room_service.py 第454-459行 (leave_room)
if new_status == "closed":
    pipe.delete(cache.room_meta_key(room_id))  # 正确删除
    pipe.delete(cache.room_users_all_key(room_id))
    pipe.delete(active_key)
    pipe.zrem(idle_zset_key, room_id)
```

#### Bug #2: AI检测接口缺少房间成员验证

**严重程度**: 中

**问题位置**: `controllers/room_controller.py` 第161-194行 `detect_person_api`函数

**问题描述**:
`detect_person_api`只检查`body.user_id != user.id`（token用户与body用户是否匹配），但没有验证发起检测的用户是否在指定的房间中。这意味着一个用户A可以检测房间X中用户B的摄像头数据（如果他们有相同的token）。

**问题代码**:
```python
# room_controller.py 第171-173行
# 验证用户身份，只能检测自己的摄像头
if body.user_id != user.id:
    return _json_err(403, 403, "无权检测其他用户", {})
# 缺少：验证 user.id 是否在 body.room_id 中
```

**修复建议**:
在检测前应验证用户是否在房间中：
```python
if body.user_id != user.id:
    return _json_err(403, 403, "无权检测其他用户", {})

# 添加房间成员验证
if not cache.is_user_active_in_room(body.room_id, user.id):
    return _json_err(403, 403, "用户不在该房间中", {})
```

### 3.2 测试数据问题（无需修改代码）

#### 测试数据问题 #1: Pydantic验证优先级

**说明**:
Pydantic对请求模型的缺失字段验证返回422（Unprocessable Entity），但项目中的`user_service.register_user`函数对空值有自己的业务逻辑验证，返回400（Bad Request）。

**原因**:
这是FastAPI/Pydantic的标准行为。当请求体完全缺失某个字段时（如只传`{"phone": "xxx"}`不传`password`），Pydantic会先于业务逻辑进行验证返回422。但当字段被设置为空字符串时（如`{"phone": "xxx", "password": ""}`），Pydantic允许空值，业务逻辑才会介入返回400。

**测试修正**:
测试用例应该测试空字符串场景而非完全缺失字段的场景，或接受422和400两种响应。

### 3.3 测试框架问题

#### 测试框架问题 #1: WebSocket测试API差异

**说明**:
TestClient使用`websocket_connect`而非`ws_connect`来建立WebSocket连接。

**测试修正**:
```python
# 错误
response = self.client.ws_connect("/ws/room/test123")

# 正确
with self.client.stream("GET", "/ws/room/test123?user_id=test") as response:
    pass
```
