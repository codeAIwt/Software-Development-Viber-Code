# 阶段一 预期成果文件

阶段：第一阶段 MVP 验证（用户模块 + 自习室接口占位）

适用范围：
- 后端：`backend`（FastAPI，v0.1.0）
- 前端：本阶段仅要求后端接口链路可用（前端若需联调可在下一阶段细化）

## 1. 环境前置

1. VSCode 已打开 `project/` 或至少打开 `project/backend/`。
2. 已在 `project/backend` 创建 venv：`.venv`。
3. VSCode 选中 Interpreter 为 `project/backend/.venv` 下的 python。
4. 后端已启动：`uvicorn app:app --reload --host 0.0.0.0 --port 8000`。

## 2. 统一响应结构（必须一致）

成功响应结构：
```json
{ "code": 200, "msg": "OK|具体业务成功信息", "data": { } }
```

失败响应结构：
```json
{ "code": <错误码>, "msg": "<错误信息>", "data": { } }
```

说明：
- 本阶段只要求在“可鉴权/可正常解析 JSON”的前提下保持统一响应。
- 自习室接口在未接入存储层时允许返回 `code=501`，但不允许返回 500/堆栈崩溃。

## 3. 用户模块用例（期望结果）

准备：
- 使用任意手机号字符串（当前实现不做手机号格式校验），如 `13900003333`
- 准备任意密码字符串（当前实现不做密码加密与强校验），如 `secret1`
- 将登录/注册接口返回的 token 记为 `${token}`

用例 U1：`POST /api/user/register`
- 期望：
  - `code == 200`
  - `msg == "注册成功"`
  - `data.user_id` 非空字符串
  - `data.token` 非空字符串

用例 U2：`POST /api/user/login`
- 期望：
  - `code == 200`
  - `msg == "登录成功"`
  - `data.token` 非空字符串

用例 U3：`GET /api/user/profile`（携带 `Authorization: Bearer ${token}`）
- 期望：
  - `code == 200`
  - `data.phone` 等于请求的手机号
  - `data.nickname` 为 `用户{手机号后四位}` 形式（做包含/后缀匹配即可）
  - `data.avatar` 为默认头像 URL（非空）

用例 U4：`PUT /api/user/profile/nickname`（携带 Authorization）
- 请求：`{ "nickname": "新昵称" }`
- 期望：
  - `code == 200`
  - `msg == "昵称更新成功"`
  - `data.nickname == "新昵称"`

用例 U5：`POST /api/user/logout`（携带 Authorization）
- 期望：
  - `code == 200`
  - `msg == "退出成功"`
  - 继续使用同一个 token 调用 `GET /api/user/profile`：
    - 期望 `code == 401`

## 4. 自习室接口用例（期望结果：接口占位）

当前自习室服务层为占位实现，按要求需“仅定义类与函数即可，暂不增加功能细节”，因此后端路由在未接入存储层时必须返回 `code=501`，而不是 500 或崩溃。

用例 R1：`POST /api/room/create`
- 期望：`code == 501`

用例 R2：`GET /api/room/list?theme=考研`（或不传 theme）
- 期望：`code == 501`

用例 R3：`POST /api/room/join`
- 期望：`code == 501`

用例 R4：`POST /api/room/leave`
- 期望：`code == 501`

## 5. 通过标准（Pass）

1. 用户模块 U1~U5 全部满足期望结果。
2. 自习室接口 R1~R4 返回 `code=501` 且响应体符合统一结构，不出现未捕获异常导致的 500。
