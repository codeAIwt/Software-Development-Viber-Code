# 二：核心功能（MVP功能）与操作流程 (Core Workflows) 【核心业务说明】
## 核心功能1：用户系统
- 页面交互与执行步骤(Step-by-Step Flow)：
1. 未登录用户进入平台任意页面，均会触发轻量弹窗提示，提供 “登录”“注册” 快捷入口，也可关闭弹窗进入游客模式（仅可浏览，无法使用自习室、时长统计等核心功能）。
2. 注册流程： 
    1. 用户点击 “注册” 进入 /register 路由页面，页面展示注册表单，包含：手机号（文本，唯一）、密码（文本，6-16 位字母数字组合）、确认密码（文本）；
    2. 用户填写所有信息后点击 “完成注册”，系统拦截校验：两次密码是否一致、密码是否符合格式；
    3. 数据操作：调用后端 API，向用户表（user）插入一条新记录，包含手机号、加密后的密码、注册时间，默认用户昵称设为 “自习小伙伴_+ 随机数字”，头像设为平台默认图；
    4. 页面反馈：注册成功后，Toast 提示 “注册成功，自动为您登录”，自动完成登录并跳转到平台首页 /home。
3. 登录流程：
    1. 用户点击 “登录” 进入 /login 路由页面，提供并展示 “手机号 + 密码”表单，包含：手机号、密码
    2. 填写信息后点击“立即登录”，系统拦截校验表单信息是否为空，随后调用后端 API 校验账号信息有效性；
    3. 数据操作：校验通过后，后端生成登录令牌（Token），关联用户 id 并设置有效期，前端将 Token 存储在本地缓存，同时拉取用户基础信息、学习数据；
    4. 页面反馈：登录成功后，Toast 提示 “登录成功”，跳转到平台首页 /home；校验失败Toast 提示“登录失败，请重试”。
4. 退出登录流程：
    1. 已登录用户在个人中心 /personal 页面，点击 “退出登录” 按钮，弹出确认弹窗 “确定要退出当前账号吗？”；
    2. 用户点击 “确认”，系统执行数据操作：前端清除本地缓存的登录 Token，后端销毁当前用户的登录状态；
    3. 页面反馈：退出成功后，Toast 提示 “已成功退出登录”，自动跳转到平台登录页 /login。

## 核心功能2：全应用自习时长激励排行榜
- 页面交互与执行步骤(Step-by-Step Flow)：
1. 用户进入个人中心 /personal 路由页面，找到 “学习数据” 模块。
2. 页面自动拉取并展示用户当日自习累计时长、日平均自习时长，以及核心展示项 “击败 XX% 全平台自习成员(调用计算函数，计算方式：按每小时分段，并统计全平台不同时间段的人数，然后统计“低于用户所在时间段的人数和”/“全平台用户数” *100%，作为展示数据)”。
3. 系统每日凌晨 4 点触发定时任务，统计全应用用户前一日的自习时长并计算平均值，基于用户个人数据匹配对应的击败百分比。
4. 数据操作：后端通过定时任务遍历用户学习时长表，计算全量用户数据后，将用户个人日平均时长、击败百分比同步至 Redis 缓存，前端从缓存拉取数据展示。
5. 页面反馈：若用户当日无自习记录，展示 “今日尚未开始学习，快去开启自习吧” 提示语，无当日击败百分比数据；有数据时实时展示统计结果，数据更新后页面自动刷新。

## 核心功能3：学习主题匹配及自习室创建 / 加入
- 页面交互与执行步骤(Step-by-Step Flow)：
1. 用户进入自习室主页面 /study-room 路由页面，可选择 “创建自习室” 或 “加入自习室” 按钮。
2. 若选择创建自习室：
    1. 弹出创建表单，包含学习主题选择（下拉框，如考研、备战期末、考公、语言学习等）、自习室最大人数设置（数字输入，默认 4 人，上限 8 人）；
    2. 用户确认创建后，系统校验主题是否选择、人数是否在合理区间；
    3. 数据操作：调用后端 API，向自习室表插入记录，状态设为 “空闲”，关联创建者 user_id 与选定学习主题标签；
    4. 系统自动从同主题的空闲用户队列中匹配对应人数的用户，推送自习室加入邀请。
3. 若选择加入自习室：
    1. 页面展示所有 “空闲” 状态的自习室列表，包含自习室主题、当前人数 / 最大人数、创建时间；
    2. 用户点击目标自习室的 “加入” 按钮，系统校验该自习室是否仍有空闲位置；
    3. 数据操作：调用后端 API，将用户 user_id 关联至该自习室表，更新自习室当前人数；
5. 页面反馈：创建 / 加入成功后，全局 Toast 提示 “操作成功”，并重定向至该自习室的在线伴学页面 /study-room/[id]；若校验失败（如人数超限、无空闲位置），Toast 提示“加入失败，请重试”。

## 核心功能4：隐私化在线伴学
- 页面交互与执行步骤(Step-by-Step Flow)：
1. 用户进入自习室在线伴学页面 /study-room/[id]，页面加载完成后弹出权限申请弹窗，申请摄像头、麦克风使用权限。
2. 用户授权后，系统默认执行：开启摄像头采集画面，关闭麦克风音频传输，将用户学习画面展示在自习室的成员视频列表中。
3. 页面提供隐私模式切换入口，位于视频画面角落，包含 “原画面”“仅展示手部”“背景模糊” 三个选项。
4. 若用户切换隐私模式：
    1. 选择 “仅展示手部”：前端对摄像头采集的视频流进行画面裁剪，仅保留手部区域展示；
    2. 选择 “背景模糊”：前端调用滤镜接口，对视频流背景进行高斯模糊处理，保留人物主体清晰；
5. 数据操作：用户的模式选择同步至后端，在自习室成员列表中标注该用户的展示模式，视频流处理均在本地完成，不传输原始未处理画面。
6. 页面反馈：权限授权成功后，视频画面正常加载；授权失败时，Toast 提示 “请开启摄像头权限以进行在线伴学”；模式切换成功后，视频画面实时更新，无额外弹窗干扰；用户可随时点击摄像头关闭按钮，关闭后视频画面展示为 “已关闭摄像头” 占位图。
