## 3. 失败测试详细分析

### 3.1 T-001 & T-002: Pydantic验证优先级问题（测试数据问题）

**问题描述**：
测试用例期望当请求体缺少必填字段时返回422状态码，但实际代码返回400。

**测试代码**：
```python
def test_register_missing_password(self):
    response = self.client.post("/api/user/register", json={"phone": "13800138000"})
    self.assertEqual(response.status_code, 422)  # 期望422
```

**实际响应**：
```json
{"code": 400, "msg": "phone 与 password 不能为空", "data": {}}
HTTP Status: 400
```

**根本原因分析**：

1. **Pydantic模型定义** (`controllers/user_controller.py` 第35-37行)：
```python
class RegisterBody(BaseModel):
    phone: str = Field(default="")
    password: str = Field(default="")
```

2. **问题根源**：
   - Pydantic使用`Field(default="")`设置默认值空字符串，而不是`Field(...)`表示必填
   - 当请求体为`{"phone": "13800138000"}`时（缺少password），Pydantic使用默认值""填充password字段
   - 请求体成功通过Pydantic验证，进入业务逻辑
   - 业务逻辑`user_service.register_user`检测到password为空字符串，抛出`UserServiceError(400, "phone 与 password 不能为空")`
   - 最终返回400

3. **测试期望与实际行为的差异**：
   - 测试期望：Pydantic验证失败返回422
   - 实际行为：Pydantic使用默认值填充，业务逻辑检测到空值返回400

**结论**：**这是测试数据问题，不是代码Bug**

- 测试期望不符合Pydantic的实际行为
- 代码行为是正确的：当字段为空字符串时返回400是合理的业务响应
- 如果期望Pydantic返回422，需要将模型定义为：
```python
class RegisterBody(BaseModel):
    phone: str = Field(...)  # 必填，无默认值
    password: str = Field(...)  # 必填，无默认值
```

### 3.2 T-003: WebSocket测试API差异（测试框架问题）

**问题描述**：
测试用例使用`ws_connect`方法连接WebSocket，但TestClient没有此方法。

**测试代码**：
```python
def test_websocket_missing_user_id(self):
    response = self.client.ws_connect("/ws/room/test123")
    response.close(code=1008, reason="Missing user_id")
```

**错误信息**：
```
AttributeError: 'TestClient' object has no attribute 'ws_connect'
```

**根本原因分析**：

1. **正确的API**：TestClient使用`websocket_connect`而非`ws_connect`
```python
# 错误
self.client.ws_connect("/ws/room/test123")

# 正确
self.client.websocket_connect("/ws/room/test123")
```

2. **修复后的测试代码**：
```python
def test_websocket_missing_user_id(self):
    with pytest.raises(WebSocketDisconnect):
        self.client.websocket_connect("/ws/room/test123")
```

**结论**：**这是测试框架问题，不是代码Bug**

- 实际WebSocket端点功能正常
- TestClient的API名称与测试代码不匹配
- 需要修改测试代码使用正确的API
