# 第二阶段测试预期（MVP 自习室模块）

## 测试目标
验证自习室模块的核心功能，包括创建、列表、加入、退出、摄像头和学习时长记录等功能。

## 测试环境
- 前端：Vue 3 + Vite 开发服务器
- 后端：Python 3.12 + FastAPI
- 缓存：Redis
- 数据库：SQLite

## 测试步骤与预期结果

### 1. 自习室创建

#### 测试步骤
1. 登录系统，进入自习室列表页面
2. 点击"创建房间"按钮
3. 选择主题（考研/期末/考公/语言）
4. 选择人数上限（1-8人）
5. 选择标签（最多3个）
6. 点击"创建"按钮

#### 预期结果
- 成功创建自习室
- 页面跳转到自习室详情页面
- 控制台打印创建者的加入时间
- 后端数据库和Redis中正确存储房间信息

### 2. 自习室列表

#### 测试步骤
1. 登录系统，进入自习室列表页面
2. 从"主题/标签"下拉框中选择一个选项
3. 观察房间列表的变化

#### 预期结果
- 页面显示符合筛选条件的空闲自习室
- 每个房间显示主题、人数、状态和标签
- 列表按最新活跃时间排序

### 3. 加入自习室

#### 测试步骤
1. 登录系统，进入自习室列表页面
2. 选择一个空闲的自习室，点击"加入"按钮

#### 预期结果
- 成功加入自习室
- 页面跳转到自习室详情页面
- 控制台打印用户的加入时间
- 房间的当前人数增加1
- 如果房间满员，状态变为"full"

### 4. 退出自习室

#### 测试步骤
1. 进入自习室详情页面
2. 点击"退出房间"按钮

#### 预期结果
- 成功退出自习室
- 页面返回自习室列表页面
- 控制台打印用户的离开时间和学习时长
- 房间的当前人数减少1
- 如果房间无人，状态变为"closed"

### 5. 摄像头功能

#### 测试步骤
1. 进入自习室详情页面
2. 观察摄像头是否启动
3. 点击"重试"按钮（如果摄像头启动失败）

#### 预期结果
- 摄像头成功启动并显示画面
- 如果浏览器不支持摄像头或没有权限，显示相应的错误提示
- 本地访问（localhost或127.0.0.1）时摄像头可以正常工作

### 6. 学习时长记录

#### 测试步骤
1. 加入自习室
2. 等待一段时间
3. 退出自习室

#### 预期结果
- 控制台打印用户在房间中的学习时长（分钟）
- 学习时长记录正确

## 测试注意事项

1. **主题选择**：创建自习室时只能选择基础主题（考研/期末/考公/语言），其他标签只能作为房间标签。

2. **标签选择**：创建自习室时最多选择3个标签，标签可以是基础主题或用户自定义标签。

3. **人数限制**：自习室人数上限必须在1-8人之间。

4. **状态变化**：
   - 创建时：如果人数上限为1，状态为"full"；否则为"idle"
   - 加入时：如果人数达到上限，状态变为"full"
   - 退出时：如果人数为0，状态变为"closed"；否则为"idle"

5. **摄像头访问**：
   - 本地访问（localhost或127.0.0.1）时可以正常访问摄像头
   - 私网访问时，浏览器可能会阻止摄像头访问，需要在浏览器中启用相关选项

6. **并发测试**：
   - 多人同时加入同一房间时，最后一个位置应该只有一人成功
   - 多人同时退出房间时，人数减少应该正确

## 问题与解决方案

1. **问题**：使用私网地址访问时摄像头无法工作
   **解决方案**：使用localhost或127.0.0.1访问，或者在浏览器中为该私网地址启用摄像头访问权限

2. **问题**：创建自习室时选择非基础主题作为主题
   **解决方案**：主题下拉框只显示基础主题，标签通过标签选择弹窗选择

3. **问题**：标签选择超过3个
   **解决方案**：前端限制最多选择3个标签，后端也有相应的验证

## 总结

本测试文档覆盖了自习室模块的核心功能，包括创建、列表、加入、退出、摄像头和学习时长记录等。通过这些测试，可以验证自习室模块是否按照设计要求正常工作。

测试过程中需要注意浏览器的安全策略对摄像头访问的影响，以及并发操作对房间状态的影响。如果测试过程中发现问题，应及时记录并解决。
