# 二、 需求背景

## 产品 / 数据现状

得益于移动互联网以及网络直播技术的发展，继线下自习室火爆之后，形式多样的线上自习室同样遍地开花。用户点击进入之后可以进行连麦，直播自己的学习画面，并实时显示学习时长。

除了直播连线这种形式，在手机应用商店检索“自习室”，可以看到大量相关App，大致可以分为两类：一类是以番茄ToDo为代表主打时间管理和学习计划的产品，另一类是以CoStudy为代表主打学习社区的产品。

根据最新的市场调研报告，中国线上自习室市场在过去几年中经历了显著的增长。据预测，到2025年，市场规模将超过100亿元人民币。这一增长主要得益于在线教育需求的激增、远程学习趋势的加速以及技术进步带来的用户体验提升。可见，线上自习室有较大的市场潜力和发展空间。

## 用户调研

用户需求：

* 基础需求：稳定的线上空间、计时功能、背景音 / 白噪音。

* 进阶需求：专注模式、学习数据统计、同伴监督、学习计划制定。

* 情感需求：归属感、成就感、避免孤独感。

核心用户：

考研/考公/考证人群，初高中生，职场人

## 4.2 功能详细说明

### 用户登录/注册

用户登录成功后，首次登录会显示标签选择弹窗，用户可以选择感兴趣的学习标签，也可以跳过。选择后的标签会显示在用户信息界面，用户可以通过点击加号按钮修改标签。

![](<images/产品需求文档 -image-1.png>)

### 自习室创建

用户在创建自习室时，需要选择自习室的主题、人数上限和标签（最多3个）。标签选择弹窗与用户首次登录时的标签选择弹窗类似，用户可以从基础主题和个人标签中选择。

### AI检测功能

为了确保用户在自习室中的真实存在，系统会每1分钟自动检测用户是否在摄像头前方。当检测到用户不在摄像头前方时，系统会自动将用户从自习室中退出。

**检测流程**：
1. 系统每1分钟自动截取摄像头画面
2. 使用AI模型分析画面中是否有人
3. 如果检测到无人，自动将用户从自习室中退出
4. 用户退出时会收到学习时长的弹窗提示

**隐私保护**：
- AI检测仅在本地进行，不上传用户图像到云端
- 检测完成后，图像数据会被立即销毁
- 检测结果仅用于判断用户是否在摄像头前，不做其他用途

**技术实现**：
- 使用ResNet-18模型进行人脸识别
- 支持在隐私模式下进行检测
- 检测失败时默认认为有人在摄像头前，避免误判



# 五、 非功能需求

> 可以列举产品营销需求、运营需求、财务需求、法务需求、使用帮助、问题反馈等

## 5.1、产品营销需求

1. 品牌调性统一

   * 产品视觉风格需简洁、安静、无干扰，符合“专注学习”定位，避免娱乐化、游戏化元素。

2. 用户增长策略支持

   * 提供可分享的学习成就卡片（如“今日学习3小时，击败85%用户”），用于社交媒体传播。

3. 数据埋点能力

   * 埋点覆盖关键路径（注册、匹配、进入自习室、退出、查看排行榜），用于分析用户留存与转化漏斗。

4. A/B测试支持

   * 支持对匹配算法策略、界面文案、激励文案等进行灰度发布与效果对比。

## 5.2、运营需求

1. 内容运营

   * 后台可配置学习主题标签（如学习主题（考研/期末/考公/语言等）），支持动态增删。

2. 活动运营

   * 支持限时学习挑战活动（如“7天连续打卡”），系统自动统计并发放虚拟徽章（非装扮类）。

3. 用户分层管理

   * 根据活跃度（日均学习时长、周登录频次）划分用户等级，用于精细化推送（如高活跃用户优先匹配）。

4. 异常行为监控

   * 摄像头遮挡、频繁进出等行为，可临时限制匹配权限。

## 5.3、法务与合规需求

1. 用户隐私保护

   * 遵守《个人信息保护法》《网络安全法》，明确告知用户视频/音频仅用于实时伴学，不存储、不转发、不留存。

   * 用户可随时关闭摄像头，且系统不得强制开启。

2. 数据最小化原则

   * 仅收集必要信息：用户名、邮箱、学习主题偏好、学习时长。

   * 不采集设备ID、通讯录、位置等无关数据。

3. 未成年人保护

   * 若用户年龄<14岁，需获得监护人同意（可通过邮箱验证+声明确认）。

4. 服务协议与隐私政策

   * 注册流程中强制用户阅读并勾选《用户协议》《隐私政策》。

5. 内容安全

   * 虽无聊天功能，但仍需防范视频画面违规（如展示不当内容），可接入AI图像审核（低频抽检）。
